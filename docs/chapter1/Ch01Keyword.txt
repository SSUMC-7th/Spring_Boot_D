- 외래키
    - 두개의 테이블을 이어주는 역할
        - 외래 키는 다른 테이블의 기본키를 나타낸다.
        - 외래 키를 가지고 있는 테이블을 하위 테이블 / 기본키를가지고 있는 테이블을 상위 테이블이라고 한다.
        - 상위 테이블의 기본키 혹은 고유키를 외래키로 지정할 수 있다.
        - 기본키에 존재하지 않는 값은 외래키가 될 수 없다.
- 기본키
    - 데이터베이스 테이블 내의 모든 레코드에서의 고유한 아이디
        - 모든 레코드는 다른 값을 가지고 있다 → 모두 다르게에 식별할 수 있다.
        - NULL은 존재할 수 없다.
        - 레코드의 값은 중복이 불가능하며 경우에 따라 1개 이상의 기본키를 정의할 수 도있다.
    
- ER 다이어그램
     Entity-Realtion Diagram : 개체 관계도
    
    개체
    - 단독으로 존재하는 객체를 의미하며, 동일한 객체는 존재하지 않는다.
    - ERD 객체 표시 방법
        
        사각형으로 표시한다.
        
    속성
    
    - 속성은 각 객체가 가지고 있는 속성을 표시한다.
    - EED 속성 표시 방법
        원으로 표시하며 객체와 연결한다. 

        이때, 기본키는 속성이름에 밑줄을 그어 표시한다
      
    관계
    
    - 객체관의 관계를 표현한다.
    - EDR 관계 표현 방법   
    
    관계성 그리기
    
- 복합 키
    
    > Composite Key 는 두 개 이상의 컬럼을 Key로 지정하는 것을 말한다.
    > 
- 연관관계
    
    두 도메인 ( 객체, 테이블 ) 이 서로 논리적인 의미를 가지고 양쪽다 참조하는 관계를 의미한다. 
    
    - 방향 : 단방향, 양방향
    - 연관관계의 주인 : 양방향 연관관계 일때 외래키를 관리하는 객체
    - 다중성 : 1:1, N:1, 1:N, N;M
- 정규화
    
    > 관계형데이터베이스의 설계에서 데이터의 중복을 줄이고 데이터 무결성을 개선하기 위해 데이터를 정규형(normal form)에 맞도록 구조화하는 프로세스를 뜻한다.
    > 
    
    데이터베이스 정규화의 목표는 이상이 있는 관계를 재구성하여 작고 잘 조직된 관계를 생성하는 것에 있다.
    
    제1정규화 : 테이블의 컬럼이 하나의 값을 가지도록 테이블을 분해하는 것
    
    제2정규화 : 완전 함수 종속을 만족하도록 테이블을 분해하는 것. 완전함수정속이란 기본키의 부분집합이 결정자가 되어선안된다는 것을 의미
    
    제3정규화 : 이행적 종속이 없도록 테이블을 분해하는 것 A→ B, B→C 라는 관계가 있을 때, A→C라는 관게가 없도록 하는 것.  
    
    BCNF 정규화 : 모든 결정자가 후보키가 되도록 테이블을 분해하는 것.
    
- 반 정규화
    
    데이터베이스의 성능향상을 위해 데이터의 중복을 허용하고 join을 줄임
    
    반정규화 데이터는 정규화 데이터에 비해 Select의 속도는 높이지만 모델의 유연성은 낮아짐.
